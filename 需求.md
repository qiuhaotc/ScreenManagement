## 1. 项目概述

### 1.1 项目背景

Windows 用户在切换显示模式（仅电脑屏幕 / 复制 / 扩展 / 仅第二屏幕）和 HDR 状态时，需要进入系统设置多层菜单操作，流程繁琐。本项目提供一个轻量级托盘工具，允许用户通过全局快捷键或托盘菜单快速切换显示模式和 HDR 状态。

### 1.2 目标用户

- 经常在外接显示器与笔记本屏幕之间切换的用户
- 使用 HDR 显示器的内容创作者、游戏玩家
- 需要批量自动化显示配置的高级用户

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|----------|
| 切换显示模式需要 4-5 步操作 | 一键全局快捷键切换 |
| HDR 开关隐藏在深层设置中 | 快捷键 / 托盘菜单直接切换 |
| 无法同时切换显示模式 + HDR | 支持组合快捷键一次完成多项操作 |
| 不清楚当前显示状态 | 托盘图标和 Tooltip 实时反映状态 |

## 2. 功能需求

### 2.1 显示模式管理

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| F-01 | 获取当前显示模式 | P0 | 通过 Windows API 查询当前显示拓扑（内部/克隆/扩展/外部） |
| F-02 | 设置显示模式 | P0 | 切换到指定显示模式（内部/克隆/扩展/外部） |
| F-03 | 显示模式快捷键 | P0 | 为每种显示模式绑定独立全局快捷键 |
| F-04 | 显示器枚举 | P0 | 枚举所有连接的显示器，获取名称、ID、分辨率、HDR 能力等 |
| F-05 | 显示器热插拔检测 | P1 | 监听 WM_DEVICECHANGE，实时更新显示器列表和状态 |

### 2.2 HDR 管理

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| F-06 | 获取 HDR 状态 | P0 | 查询指定显示器的 HDR 启用状态 |
| F-07 | 设置 HDR 状态 | P0 | 切换指定显示器的 HDR 开关 |
| F-08 | HDR 快捷键 | P0 | 为每台显示器绑定 HDR 切换快捷键 |
| F-09 | HDR 能力检测 | P0 | 检测显示器是否支持 HDR，不支持则禁用相关设置项 |

### 2.3 组合动作

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| F-10 | 组合快捷键 | P1 | 一键执行"切换显示模式 + 切换 HDR"等组合操作 |
| F-11 | 自定义动作序列 | P2 | 用户可自定义多个显示动作按顺序执行 |

### 2.4 系统托盘

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| F-12 | 托盘图标 | P0 | 显示在系统托盘中，图标反映当前显示模式（不同图标表示不同模式） |
| F-13 | 托盘 Tooltip | P0 | 鼠标悬停显示当前显示模式和 HDR 状态摘要 |
| F-14 | 托盘右键菜单 | P0 | 右键菜单包含：快速切换显示模式、切换 HDR、打开主界面、退出 |
| F-15 | 托盘左键行为 | P1 | 左键单击打开主界面；可配置为直接切换预设模式 |

### 2.5 开机自启

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| F-16 | 开机自启注册 | P1 | 通过注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 或任务计划程序注册 |
| F-17 | 静默启动 | P1 | 开机启动时不显示主窗口，直接最小化到托盘 |
| F-18 | 自启开关 | P1 | 在设置界面提供开机自启的启用/禁用开关 |

### 2.6 快捷键管理

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| F-19 | 全局快捷键注册 | P0 | 使用 `RegisterHotKey` Windows API 注册全局热键 |
| F-20 | 快捷键冲突检测 | P1 | 注册失败时提示用户快捷键已被占用，要求更换 |
| F-21 | 快捷键持久化 | P0 | 快捷键配置保存到本地 JSON 文件，启动时加载 |
| F-22 | 修饰键支持 | P0 | 支持 Ctrl / Alt / Shift / Win 修饰键组合 |

## 3. 非功能需求

### 3.1 性能

- 快捷键响应延迟 < 200ms
- 启动时间 < 2s（冷启动）
- 内存占用 < 100MB（空闲状态）
- CPU 占用 < 1%（空闲状态，除全局钩子外）

### 3.2 可靠性

- 快捷键注册失败时有降级策略（仅托盘操作可用）
- API 调用失败时给出用户友好的错误提示，不崩溃
- 显示器热插拔后自动刷新状态，不残留无效显示器

### 3.3 兼容性

- 操作系统：Windows 10 2004+ / Windows 11
- 架构：x64（优先）、ARM64（后续考虑）
- .NET Runtime：.NET 10 Desktop Runtime

### 3.4 安全性

- 不请求管理员权限（显示切换和 HDR 切换无需提权）
- 配置文件存储在用户目录，不含敏感信息
- 无网络请求（除检查更新外，可选功能）

### 3.5 可维护性

- 严格遵循 IoC / DI 原则，接口与实现分离
- 核心逻辑（Business 层）零 UI 依赖，完全可单元测试
- 使用 MVVM 模式组织 UI 层代码

## 4. 数据模型

### 4.1 枚举定义

```csharp
// 显示模式（对应 Windows DisplayTopology）
public enum DisplayMode
{
    Internal = 0,    // 仅电脑屏幕
    Clone = 1,       // 复制
    Extend = 2,      // 扩展
    External = 3     // 仅第二屏幕
}

// 快捷键动作类型
public enum HotkeyActionType
{
    SetDisplayMode,       // 切换显示模式
    ToggleHdr,            // 切换 HDR
    CompositeAction       // 组合动作
}
```

### 4.2 核心模型

```csharp
// 显示器信息
public class DisplayInfo
{
    public string DeviceId { get; set; }        // 设备唯一标识
    public string DisplayName { get; set; }     // 显示名称（如 "DELL U2723QE"）
    public bool IsPrimary { get; set; }         // 是否为主显示器
    public bool IsInternal { get; set; }        // 是否为内置显示器（笔记本屏幕）
    public bool SupportsHdr { get; set; }       // 是否支持 HDR
    public bool HdrEnabled { get; set; }        // HDR 当前是否启用
    public Rect Bounds { get; set; }            // 显示器区域
    public int ResolutionX { get; set; }
    public int ResolutionY { get; set; }
    public uint RefreshRate { get; set; }
}

// 快捷键绑定
public class HotkeyBinding
{
    public string Id { get; set; }              // 唯一标识 (GUID)
    public HotkeyActionType ActionType { get; set; }
    public DisplayMode? TargetMode { get; set; }// 目标显示模式（SetDisplayMode 时使用）
    public string TargetDisplayId { get; set; } // 目标显示器 ID（ToggleHdr 时使用）
    public bool? HdrTargetState { get; set; }   // HDR 目标状态（null=切换）
    public ModifierKeys Modifiers { get; set; } // 修饰键
    public uint Key { get; set; }              // 虚拟键码
    public bool IsEnabled { get; set; }         // 是否启用

    // 组合动作相关
    public List<HotkeyBinding> SubActions { get; set; }
    public bool IsComposite => ActionType == HotkeyActionType.CompositeAction;
}

// 应用配置
public class AppConfig
{
    public List<HotkeyBinding> HotkeyBindings { get; set; } = new();
    public bool AutoStart { get; set; } = false;
    public bool StartMinimized { get; set; } = true;
    public bool ShowNotifications { get; set; } = true;  // 是否显示切换通知
    public string Language { get; set; } = "zh-CN";
}
```

## 5. Windows API 映射

### 5.1 显示模式切换

| API | 用途 |
|-----|------|
| `SetDisplayConfig` (`user32.dll`) | 设置显示拓扑（内部/克隆/扩展/外部） |
| `QueryDisplayConfig` (`user32.dll`) | 查询当前显示拓扑和模式 |
| `GetDisplayConfigBufferSizes` (`user32.dll`) | 获取 QueryDisplayConfig 所需缓冲区大小 |
| `DisplayConfigGetDeviceInfo` (`user32.dll`) | 获取显示器设备详细信息 |

**关键结构体**：
- `DISPLAYCONFIG_PATH_INFO`
- `DISPLAYCONFIG_MODE_INFO`
- `DISPLAYCONFIG_TOPOLOGY_ID`（`DISPLAYCONFIG_TOPOLOGY_INTERNAL` / `CLONE` / `EXTEND` / `EXTERNAL`）

### 5.2 HDR 管理

| API | 用途 |
|-----|------|
| `D3DKMTSetVidPnSourceHdrMetaData` (`gdi32.dll`) | 通过 GDI 内核模式设置 HDR 元数据（切换 HDR） |
| `GetMonitorCapabilities` (`dxva2.dll`) | 获取显示器能力（含 HDR 支持检测） |

> **备选方案**: 如果 GDI 内核模式 API 在用户态下不可用，可考虑通过修改注册表 `HKLM\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\MonitorDataStore` 或使用 `DisplayConfigSetDeviceInfo` 设置颜色信息。

### 5.3 显示器枚举

| API | 用途 |
|-----|------|
| `EnumDisplayMonitors` (`user32.dll`) | 枚举所有显示器 |
| `GetMonitorInfo` (`user32.dll`) | 获取显示器详细信息 |
| `EnumDisplayDevices` (`user32.dll`) | 枚举显示设备（含内部/外部标识） |
| `MonitorFromWindow` / `MonitorFromPoint` | 从窗口/坐标获取显示器句柄 |

### 5.4 全局快捷键

| API | 用途 |
|-----|------|
| `RegisterHotKey` (`user32.dll`) | 注册全局热键 |
| `UnregisterHotKey` (`user32.dll`) | 注销全局热键 |
| `WM_HOTKEY` (0x0312) | 窗口消息，热键触发时接收 |

### 5.5 显示器变更通知

| API/消息 | 用途 |
|-----|------|
| `WM_DISPLAYCHANGE` | 显示器分辨率/色彩深度变化 |
| `RegisterDeviceNotification` + `WM_DEVICECHANGE` | 显示器连接/断开检测 |

### 5.6 开机自启

| 方式 | 说明 |
|------|------|
| 注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` | 用户级自启，首推方式 |
| 任务计划程序 | 备选，可实现延迟启动等高级控制 |

## 7. 技术选型

| 层次 | 技术 | 理由 |
|------|------|------|
| 运行时 | .NET 10 | 最新 LTS，性能优秀，Windows 互操作成熟 |
| UI 框架 | **WPF** (.NET 10) | 原生 Windows UI，成熟的 MVVM 支持，托盘/热键原生集成，Fluent Design 扩展 |
| DI 容器 | `Microsoft.Extensions.DependencyInjection` | 内置，轻量，支持泛型 Host |
| 配置存储 | `System.Text.Json` | 内置高性能 JSON 序列化 |
| 日志 | `Microsoft.Extensions.Logging` | 与 DI 集成，支持文件/控制台输出 |
| 全局热键 | Windows `RegisterHotKey` API | 系统原生，稳定可靠 |
| 显示切换 | Windows `SetDisplayConfig` API | 系统原生 CCD API |
| HDR 切换 | `D3DKMTSetVidPnSourceHdrMetaData` | 系统 GDI 内核态接口 |
| MVVM 框架 | CommunityToolkit.Mvvm | 轻量，源生成器减少样板代码 |
| 测试框架 | xUnit + Moq + FluentAssertions | 主流 .NET 测试组合 |
| CI/CD | GitHub Actions | 免费，与 GitHub 深度集成 |
| 打包 | `dotnet publish` + 单文件发布 | 生成独立 exe |

## 10. 约束与假设

### 约束

1. **仅 Windows**: 初期仅支持 Windows 10 2004+ / Windows 11，WPF 原生支持
2. **用户态运行**: 不请求管理员权限，所有操作在用户态完成
3. **单实例运行**: 同一时间仅允许一个程序实例运行（通过 Mutex 检测）
4. **HDR 限制**: HDR 切换仅在支持 HDR 的显示器上生效

### 假设

1. `SetDisplayConfig` API 在目标 Windows 版本上可用且行为一致
2. `D3DKMTSetVidPnSourceHdrMetaData` 可在用户态调用（需实际验证）
3. 用户有基本的快捷键概念认知
4. WPF 对 Windows 桌面开发有成熟支持

### 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| HDR API 在用户态不可用 | HDR 功能无法实现 | 准备注册表修改方案作为备选 |
| WPF 托盘需要 WinForms 兼容层 | 增加项目复杂度 | 使用 `System.Windows.Forms.NotifyIcon`，成熟稳定 |
| 快捷键与其他软件冲突 | 用户体验差 | 提供冲突检测 + 手动更换引导 |
| `SetDisplayConfig` 在新版本 Windows 行为变化 | 显示切换失败 | 添加异常捕获 + 日志记录 + 友好错误提示 |
