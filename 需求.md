# Windows 屏幕管理软件

Windows 屏幕管理, 支持屏幕显示快捷键切换, HDR快捷键切换

## 需求

- 支持屏幕显示快捷键切换 (仅电脑屏幕, 复制, 拓展, 仅第二屏幕.), 支持对不同的显示方式设置不同的快捷键. 注意: 可能还有更多模式.
- 支持HDR快捷键切换, 支持通过快捷键切换指定的显示器的HDR状态.
- 支持一个快捷键同时做到上述两点, 例如: 切换到复制模式的同时打开HDR.
- 支持在系统托盘显示当前的显示模式和HDR状态, 方便用户查看当前状态.
- 支持配置开机自启, 当通过开机自动启动时, 不显示主界面, 直接在系统托盘运行.

### 界面设计

界面设计应该简洁明了, 界面显示现代化, 美观且易于使用. 主要界面包括:

- 主界面: 显示当前的显示模式和HDR状态, 提供快捷键设置入口. 支持显示器列表, 以及每个显示器的当前状态 (例如: 是否支持HDR, 当前的显示模式等). 用户可以通过点击显示器来切换显示模式和HDR状态, 也可以通过快捷键来切换.
- 快捷键设置界面: 允许用户为不同的显示模式和HDR状态设置快捷键, 支持自定义快捷键组合. 注意: 需要考虑快捷键冲突的情况, 例如用户设置了一个快捷键, 但是这个快捷键已经被系统或者其他软件占用, 需要提示用户并要求重新设置. 还需要能够检测到显示器是否支持HDR, 如果不支持, 则不允许用户设置HDR相关的快捷键.
- 在系统托盘显示当前的显示模式和HDR状态, 方便用户查看当前状态. 用户可以通过右键点击系统托盘图标来快速切换显示模式和HDR状态, 以及访问设置界面.
- 关于界面: 显示软件的版本信息, 开发者信息, 以及使用说明. 连接到GitHub仓库以获取更多信息和支持. Github链接: [https://github.com/qiuhaotc/ScreenManagement](https://github.com/qiuhaotc/ScreenManagement)

### 技术选型

- 编程语言: C# (Windows平台开发较为方便)
- 框架: .NET 10, MAUI (跨平台开发, 目前只需要支持Windows)
- 屏幕管理/HDR切换: 使用Windows API
- 快捷键监听: 在程序启动后注册全局快捷键, 监听用户的快捷键输入
- 系统托盘: 使用标准的系统托盘实现方式
- 系统设计按照IOC原则, 方便后续维护和扩展. 例如, 可以将屏幕管理和快捷键监听的逻辑抽象成接口, 通过依赖注入来实现解耦. 也方便测试, 可以通过mock来测试核心逻辑的正确性 (关于屏幕显示切换和HDR, 只需要通过mock来测试, 不需要实际操作屏幕).

### 项目结构

- `Business/ScreenManagement.Business.csproj`: 切换显示模式和HDR状态的核心逻辑, 快捷键绑定功能
- `UI/ScreenManagement.UI.csproj`: 用户界面, 包括主界面, 快捷键设置界面, 关于界面, 系统托盘图标和菜单, 开机自启功能
- `Test/ScreenManagement.Test.csproj`: 单元测试项目, 用于测试核心逻辑的正确性 (关于屏幕显示切换和HDR, 只需要通过mock来测试, 不需要实际操作屏幕)
- `ScreenManagement.sln`: 解决方案文件, 包含上述三个项目
- `README.md`: 项目说明文件, 包含项目简介, 功能列表, 技术选型, 使用说明等信息
- "README.en.md": 英文版项目说明文件, 包含与README.md相同的信息, 但使用英文撰写, 方便国际用户了解和使用该软件, 和`README.md`互相链接, 方便用户切换语言版本
- `.github/acrions`: GitHub Actions配置文件, 用于自动化构建和测试(每次有commit到main branch就触发), 以及发布新版本(每次有新的tag就触发, tag格式为`vX.X.X`, 例如`v1.0.0`或者`v0.1`)
